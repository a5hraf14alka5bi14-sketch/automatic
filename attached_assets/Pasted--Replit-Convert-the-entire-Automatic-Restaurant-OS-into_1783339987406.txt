@Replit

Convert the entire **Automatic Restaurant OS** into installable native applications while preserving all existing functionality.

### 1. Mobile Apps

Create production-ready applications for:

* iOS (iPhone & iPad)
* Android (phones & tablets)

Requirements:

* Use a modern cross-platform framework (Flutter or React Native with Expo, whichever best fits the current architecture).
* Connect to the existing backend and database without changing business logic.
* Responsive UI optimized for touch devices.
* Offline caching where appropriate.
* Push notification support.
* Camera support for barcode/QR scanning.
* Secure authentication and session handling.
* Native splash screen and app icon.
* Deep linking support.
* Prepare builds for App Store and Google Play.

### 2. Windows Desktop App

Create a native Windows desktop application using Electron or Tauri (choose the best option based on performance and maintainability).

Requirements:

* Preserve all existing features.
* Native menus, desktop window management, and notifications.
* Auto-update support.
* Optimized performance.
* Installer (.exe/MSIX) ready for deployment.

### 3. Codebase

* Reuse as much of the existing code as possible.
* Do not change the application's business logic.
* Keep one shared backend.
* Maintain feature parity across Web, Mobile, and Windows.

### 4. Quality Assurance

* Test all screens on mobile, tablet, and desktop.
* Fix platform-specific UI issues.
* Ensure all existing tests pass.
* Add platform-specific tests where necessary.

### 5. Deliverables

Provide:

* iOS project
* Android project
* Windows desktop project
* Build instructions
* Deployment guide
* Summary of architectural changes
* List of any remaining limitations

The goal is to make Automatic Restaurant OS available as:

* Web Application
* iOS App
* Android App
* Windows Desktop Application

while maintaining a single, maintainable codebase and production-quality performance.
